RemoteDesk · Pack 31 — Session Recording Capture +
Encrypted Storage + Playback (real code)
 Codex / Claude Code handoff. NEW feature area (the trust/consent/receipt system, Packs 15-30, is complete).
 This pack adds session recording: the desktop captures a session as time-stamped chunks, the API stores
 chunk metadata + presigned upload refs (no media bytes through the API), and the web dashboard plays a
 recording back from an ordered manifest. Recording is consent-gated by the existing device policy + session
 approval, and recordings are content-addressed + integrity-checked so a tampered chunk is detectable on
 playback. Build order: shared → api → web → desktop → tests.
 Scope: capture → store metadata → play back, with consent gating and per-chunk integrity. Recording
 requires an explicit policy flag + consent (fail-closed: off by default). Media bytes go to object storage
 via presigned URLs, never through the API. Chunk manifest is hash-chained so reordering/tampering is
 detectable. No raw input payloads/keystrokes in any recording metadata. No real storage credentials
 in code.

packages/shared/src/recording/chunkManifest.ts — SAFE_DIRECT_COPY
Pure hash-chain over ordered chunk refs (reuse the Pack 22 chaining idea): buildManifest(chunks, sha256Hex) → {
entries, rootHash } ; verifyManifest(manifest, sha256Hex) → { valid, firstBrokenIndex } . Reordering, insertion, or a
swapped chunk sha breaks the chain. Empty list → root = genesis (valid).


packages/shared/src/recording/recordingPolicy.ts — SAFE_DIRECT_COPY
Fail-closed gate reusing the device-policy concepts: canRecord(policy, approval) denies unless a new
recordingEnabled policy flag is true AND session approval is satisfied (Pack 16). recordingDenialReason(...) for UI.
Off by default.


packages/shared/src/recording/schema.ts — SAFE_DIRECT_COPY
Zod: startRecordingSchema { sessionId, codec } ; appendChunkSchema { index>=0, sha256 (64 hex), sizeBytes>0,
durationMs>=0, storageKey } ; finalizeRecordingSchema { rootHash } .

Safety constraints
 • Recording is off by default; requires recordingEnabled policy flag and satisfied session approval (fail-
   closed).

   Media bytes go object-storage→presigned-URL only; never through the API.
 • Chunk manifest is hash-chained; playback (web) and finalize (API) both verify it and refuse a broken chain.
 • No raw input payloads, keystrokes, or clipboard contents in recording metadata (recording is screen
   media only; metadata is index/hash/size/duration).

 • No real storage credentials in code; LocalDevPresigner is a marked dev stub that fails closed without
   storage env.

 • Native OS input execution remains disabled/no-op.
