# Support Escalation Runbook

## Escalation Levels

### L1: Support Agent
- First response
- Common issues
- Account questions
- SLA: 4 hours

### L2: Technical Support
- Complex issues
- Bug investigation
- API questions
- SLA: 1 business day

### L3: Engineering
- Confirmed bugs
- Performance issues
- Integration help
- SLA: 2 business days

### L4: Engineering Team
- Critical bugs
- Security issues
- Architecture questions
- SLA: As needed

## Escalation Criteria
| Situation | Escalate To |
|-----------|-------------|
| Security concern | L4 immediately |
| Data loss | L4 immediately |
| Service outage | L3 immediately |
| Bug confirmed | L3 |
| Enterprise customer unhappy | L2 + manager |
| Legal question | Legal team |

## Escalation Process
1. Document all troubleshooting steps
2. Collect logs/screenshots
3. Create internal ticket
4. Assign to next level
5. Notify customer of escalation
6. Set follow-up reminder

## Communication
- Customer: Updated every 24 hours
- Internal: Slack #support-escalations
- Status: Updated in ticket

## De-escalation
- Issue resolved
- Workaround provided
- Customer satisfied
- Documentation updated
