# Security Incident Response Plan

## Severity Levels
| Level | Description | Examples | Response Time |
|-------|-------------|----------|---------------|
| P1 | Critical | Data breach, RCE | 15 minutes |
| P2 | High | Unauthorized admin access | 1 hour |
| P3 | Medium | Suspicious activity | 4 hours |
| P4 | Low | Policy violation | 24 hours |

## Response Steps
1. Detect -> Alert on-call engineer
2. Triage -> Classify severity
3. Contain -> Isolate affected systems
4. Eradicate -> Remove threat
5. Recover -> Restore normal operations
6. Lessons Learned -> Post-incident review
