# RemoteDesk Security Best Practices

## For All Users

### Authentication
- Use strong, unique passwords
- Enable two-factor authentication (2FA)
- Do not share your login credentials
- Log out on shared computers
- Use password manager

### Sessions
- Verify who is connecting before accepting
- Only enable permissions you need
- End sessions when not needed
- Do not leave sessions unattended
- Review your session history

### Desk ID
- Treat your Desk ID like a phone number
- Share only with trusted contacts
- Be cautious with public channels
- Remember: anyone with your Desk ID can request connection

### Network
- Use secure Wi-Fi (WPA3)
- Avoid public Wi-Fi without VPN
- Corporate networks may have restrictions
- Report suspicious connection requests

## For Organizations

### Policies
- Require approval for all sessions
- Disable unattended access by default
- Enable session recording for compliance
- Set session timeout limits
- Regular access reviews

### Admin Actions
- Enforce 2FA for all users
- Configure IP whitelisting
- Monitor audit logs
- Disable inactive accounts
- Regular security training

### Incident Response
1. End suspicious sessions immediately
2. Change affected passwords
3. Review audit logs
4. Report to security team
5. Follow organization incident procedure
