# RemoteDesk Expansion Playbook

## Expansion Signals
| Signal | Action |
|--------|--------|
| License utilization > 90% | Propose additional seats |
| Multiple teams interested | Propose enterprise plan |
| High feature adoption | Introduce advanced features |
| Positive NPS + reference | Ask for case study + expansion |
| New use cases identified | Propose additional modules |
| Executive engagement | Sponsor program |

## Expansion Plays

### Seat Expansion
1. Identify teams approaching limit
2. Show usage data
3. Offer trial of additional seats
4. Volume discount proposal
5. Close

### Feature Expansion
1. Map current features used
2. Identify unused high-value features
3. Demo relevant features
4. Pilot program
5. Measure impact
6. Close

### Org-Wide Rollout
1. Identify champion department
2. Document success story
3. Present to other departments
4. Phased rollout plan
5. Executive alignment
6. Close

## Objection Handling
| Objection | Response |
|-----------|----------|
| "Too expensive" | ROI calculator, phased approach |
| "Not enough users" | Department-level pricing |
| "Current tool works" | Side-by-side comparison |
| "No budget" | Next fiscal year planning |
