# Defect Lifecycle

```
New -> Triage -> Assigned -> In Progress -> Fixed -> Verified -> Closed
                                    |                           |
                                    v                           v
                              Rejected                    Reopened
```

## Severity
| Level | Definition | SLA |
|-------|-----------|-----|
| Critical | System unusable | Fix in 24h |
| High | Major feature broken | Fix in 3 days |
| Medium | Workaround exists | Fix in 1 week |
| Low | Cosmetic | Fix in 1 month |

## Priority
P0: Block release
P1: Fix before GA
P2: Fix in next sprint
P3: Backlog
