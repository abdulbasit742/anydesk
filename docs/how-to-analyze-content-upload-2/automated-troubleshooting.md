# RemoteDesk AI Automated Troubleshooting

This document details the functionality and implementation of the AI Automated Troubleshooting system within RemoteDesk, designed to assist support agents in quickly diagnosing and resolving issues during remote sessions.

## Overview
The Automated Troubleshooting system leverages AI-driven insights to suggest relevant troubleshooting guides and, where appropriate, automatically execute diagnostic or corrective steps on the remote system. This significantly reduces the time to resolution, improves consistency in support processes, and empowers agents with intelligent assistance.

## Features
- **Insight-Driven Suggestions**: Automatically suggests relevant troubleshooting guides based on detected session insights (e.g., CPU spikes, network issues).
- **Guided Troubleshooting Steps**: Provides a structured sequence of steps for agents to follow, including manual and automated actions.
- **Automated Step Execution**: For certain diagnostic or corrective steps, the system can execute commands on the remote system, with or without agent confirmation.
- **Configurable Automation Levels**: Administrators can define thresholds for when suggestions are made and when automated steps can be executed.
- **Knowledge Base Integration**: Guides can link to external knowledge base articles for more detailed information.

## Implementation Details

### Data Transfer Objects (DTOs)
- **`TroubleshootingStepType`**: An enum defining various types of troubleshooting actions (e.g., `CHECK_NETWORK`, `RESTART_SERVICE`, `RUN_DIAGNOSTICS`).
- **`TroubleshootingStep`**: Describes a single step in a troubleshooting guide, including its type, description, an optional command for automated execution, and flags for automation and confirmation requirements.
- **`TroubleshootingGuide`**: Represents a complete troubleshooting guide, including its title, description, the type of insight it addresses, and a list of `TroubleshootingStep`s.
- **`AutomatedTroubleshootingConfig`**: Configuration settings for the system, such as `enabled`, `autoSuggestThreshold` (severity level for suggestions), and `autoExecuteThreshold` (severity level for auto-execution).
- **Location**: `remotedesk/packages/shared/src/ai/troubleshooting.dto.ts`

### API Service Logic
- **`AutomatedTroubleshootingService.ts`**: Manages troubleshooting guides, suggestions, and automated step execution on the API server.
  - **Configuration Management**: Loads and updates automated troubleshooting settings.
  - **Guide Management**: Loads and stores predefined `TroubleshootingGuide`s, potentially from a database or configuration files.
  - **Suggestion Engine**: Based on a `SessionInsight`, it filters and returns relevant `TroubleshootingGuide`s that match the insight type and meet the `autoSuggestThreshold`.
  - **Automated Execution**: Provides a method (`executeTroubleshootingStep`) to run predefined commands on the remote system, respecting `isAutomated` and `requiresConfirmation` flags.
- **Location**: `remotedesk/apps/api/src/ai/AutomatedTroubleshootingService.ts`

## Usage

### Configuration
1. **Enable Automated Troubleshooting**: In the RemoteDesk admin panel, enable the feature.
2. **Configure Thresholds**: Set the `autoSuggestThreshold` (e.g., Warning) for when troubleshooting guides should be suggested, and `autoExecuteThreshold` (e.g., Critical) for when automated steps can be executed (with confirmation).
3. **Define Guides**: Create and manage `TroubleshootingGuide`s, linking them to specific `SessionInsightType`s and defining their steps.

### Agent Workflow
1. A `SessionInsight` (e.g., `CPU_SPIKE`) is generated by the `SessionInsightsService`.
2. The `AutomatedTroubleshootingService` suggests relevant `TroubleshootingGuide`s to the support agent in the RemoteDesk console.
3. The agent reviews the suggested guide and can choose to:
    - Follow manual steps.
    - Initiate automated steps (if `isAutomated` is true), confirming if `requiresConfirmation` is true.
4. The system executes the automated commands on the remote desktop via the desktop agent.
5. The outcome of the steps is reported back to the agent.

## Technical Considerations
- **Security of Automated Commands**: Strict validation and sandboxing of commands executed on the remote system are paramount to prevent abuse or unintended consequences.
- **Desktop Agent Integration**: Requires a robust communication channel and execution environment on the desktop client to receive and execute commands from the API service.
- **User Confirmation**: For potentially disruptive automated steps (e.g., restarting services, rebooting), explicit agent confirmation is crucial.
- **Dynamic Guides**: The ability to dynamically update or create new troubleshooting guides without code deployments.
- **Contextual Awareness**: Enhancing guides with more context from the session (e.g., specific process names, error codes).

## Future Enhancements
- **Machine Learning for Guide Ranking**: Use ML to rank troubleshooting guides based on historical success rates and session context.
- **Natural Language Processing (NLP)**: Allow agents to describe an issue in natural language, and the system suggests guides.
- **Automated Remediation Workflows**: Define complex, multi-step automated workflows that can be triggered by specific insights.
- **Integration with Script Libraries**: Allow integration with existing enterprise script libraries for automated actions.
