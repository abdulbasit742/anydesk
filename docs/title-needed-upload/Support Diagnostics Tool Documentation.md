# Support Diagnostics Tool Documentation

This document outlines the design and functionality of the RemoteDesk Support Diagnostics Tool. This tool is designed to automate the collection of critical system and application information, making it easier for users to provide necessary data to support teams for efficient troubleshooting.

## 1. Overview

The Support Diagnostics Tool is a standalone utility or an integrated feature within the RemoteDesk desktop application that simplifies the process of gathering diagnostic data. Instead of manually guiding users through various system settings and log file locations, the tool automates this process, packages the information, and prepares it for submission to support.

## 2. Key Features

*   **Automated System Information Collection:** Gathers details about the operating system, hardware, network configuration, and installed drivers.
*   **Application Log Aggregation:** Locates and bundles all relevant RemoteDesk application logs.
*   **WebRTC Internals Dump (for Web Client):** Provides instructions or automates the capture of `chrome://webrtc-internals` data.
*   **Network Connectivity Tests:** Performs basic network tests (e.g., ping to STUN/TURN servers, port reachability).
*   **Configuration File Export:** Exports relevant RemoteDesk configuration files (redacting sensitive information).
*   **Problem Description Input:** Allows the user to input a description of their issue directly into the diagnostic report.
*   **Anonymization/Redaction:** Automatically redacts sensitive user information (e.g., passwords, PII) from logs and configuration files.
*   **Package and Export:** Compresses all collected data into a single, encrypted archive (e.g., `.zip` or `.tar.gz`) for easy submission.
*   **Upload Option:** Optionally provides a direct upload mechanism to a secure support portal or cloud storage.

## 3. Data Collected by the Tool

The tool will collect the following categories of information:

### 3.1. System Information

*   **Operating System:** Name, version, build number (e.g., Windows 11 23H2, macOS Sonoma 14.2, Ubuntu 22.04 LTS).
*   **CPU:** Model, core count, clock speed.
*   **RAM:** Total installed memory.
*   **GPU:** Model, driver version.
*   **Disk Space:** Free space on primary drive.
*   **Network Adapters:** List of active network interfaces, their IP addresses, and MAC addresses.

### 3.2. RemoteDesk Application Information

*   **Application Version:** Version of the RemoteDesk client/host application.
*   **Configuration Files:** (e.g., `settings.json`, `config.ini`) with sensitive data redacted.
*   **Log Files:** All log files generated by the RemoteDesk application, typically from the last 7 days.
*   **Crash Dumps:** If any application crashes occurred.

### 3.3. Network Diagnostics

*   **IP Configuration:** `ipconfig` (Windows) or `ifconfig`/`ip a` (Linux/macOS) output.
*   **Route Table:** `route print` (Windows) or `netstat -rn` (Linux/macOS) output.
*   **DNS Server Information:** Configured DNS servers.
*   **Ping/Traceroute Results:** To RemoteDesk backend servers, STUN/TURN servers.
*   **Port Scan Results:** Basic check for open ports required by WebRTC.

### 3.4. WebRTC Specific Data (for Web Client)

*   **`chrome://webrtc-internals` Dump:** A JSON dump of the WebRTC session statistics and events.

## 4. User Workflow

1.  **Launch Tool:** User launches the diagnostics tool (either standalone or from within the RemoteDesk application).
2.  **Describe Issue:** User provides a brief description of the problem they are experiencing.
3.  **Select Data:** User can optionally select which categories of data to include (e.g., exclude network info if it's a UI bug).
4.  **Collect Data:** The tool automatically gathers the selected information.
5.  **Review (Optional):** User can review the collected data (with redactions) before packaging.
6.  **Package Data:** The tool compresses and encrypts the data into a single file.
7.  **Submit/Save:** User can either upload the package directly to RemoteDesk support or save it locally to their computer.

## 5. Security and Privacy

*   **Data Redaction:** All personally identifiable information (PII) and sensitive credentials (e.g., passwords, API keys) will be automatically redacted or anonymized before packaging.
*   **Encryption:** The diagnostic package will be encrypted to protect data in transit and at rest.
*   **Access Control:** Access to uploaded diagnostic data will be strictly limited to authorized support and engineering personnel.
*   **Consent:** Users will be explicitly asked for consent before any data is collected and submitted.

## 6. Development Considerations

*   **Cross-Platform:** The tool should ideally be cross-platform (Windows, macOS, Linux) to provide a consistent experience.
*   **Minimal Dependencies:** Keep external dependencies to a minimum to ensure the tool is lightweight and reliable.
*   **Error Handling:** Robust error handling to gracefully manage situations where certain data cannot be collected.
*   **Version Control:** The tool should be versioned alongside the main application to ensure compatibility.
